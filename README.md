# Turing Machine Simulator


Інтерактивний візуальний симулятор машини Тюрінга, у якому можна:
- задавати правила переходів у текстовому, табличному або графовому режимі;
- запускати виконання покроково або в автоматичному режимі;
- спостерігати за стрічкою, станом, позицією головки та кількістю кроків;
- моделювати алгоритми і перевіряти логіку переходів у наочному вигляді.

## Ідея проєкту

Цей проєкт створений як навчальний інструмент для розуміння того, як працює машина Тюрінга на практиці.

Замість сухої теорії користувач отримує повноцінний візуальний стенд:
- будує автомати зі станами та переходами;
- бачить, як змінюється стрічка на кожному кроці;
- швидко перевіряє коректність правил;
- порівнює подання одних і тих самих правил у різних форматах.

Проєкт підходить для:
- студентів і викладачів курсів з теорії обчислень;
- самостійного навчання;
- демонстрацій на парах та воркшопах;
- швидкого прототипування задач на машини Тюрінга.

## Основні можливості

- Симуляція стрічки та головки у реальному часі.
- Керування виконанням: `Старт`, `Пауза`, `Крок`, `Скинути`.
- Налаштування швидкості виконання.
- Три режими редагування правил:
  - `Текст`
  - `Таблиця`
  - `Граф`
- У граф-редакторі:
  - додавання станів;
  - створення переходів між станами;
  - підтримка кратних ребер;
  - підтримка петель (`qX -> qX`);
  - ручне зміщення вигину ребер/петель для уникнення перетинів.

## Формат правил

Базовий формат рядка:

```text
{Символ, Стан} -> {НовийСимвол, Напрямок, НовийСтан}
```

Приклад:

```text
{1, q0} -> {0, R, q1}
{_, q1} -> {_, N, qf}
```

Позначення:
- `_` - порожня клітинка
- `L` - рух вліво
- `R` - рух вправо
- `N` - без руху
- `qf` - фінальний (зупиняючий) стан

## Призначення інтерфейсу

- **Стрічка**: відображає значення клітинок і поточну позицію головки.
- **Налаштування**: стартова стрічка та початковий стан.
- **Правила переходів**: редактор логіки машини в одному з трьох режимів.
- **Панель керування**: запуск, покрокове виконання, зупинка та скидання.

## Типовий сценарій роботи

1. Ввести стартову стрічку (наприклад, `1011`).
2. Вказати стартовий стан (наприклад, `q0`).
3. Додати правила переходів.
4. Запустити виконання кнопкою `Старт` або перевіряти `Крок` за `Крок`.
5. Проаналізувати результат на стрічці та в поточному стані.
